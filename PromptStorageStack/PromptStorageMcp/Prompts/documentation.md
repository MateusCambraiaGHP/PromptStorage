# Documentation Generation Rules

## XML Documentation
- **NEVER** write XML summary comments (`/// <summary>`) on code
- Code should be self-explanatory through good naming and structure

## README Structure
When generating a README, follow this structure:

1. **Title** — project name
2. **Description** — one paragraph explaining what it does
3. **Getting Started** — prerequisites, installation, configuration
4. **Usage** — code examples
5. **API Reference** — link or summary of endpoints/methods
6. **Contributing** — how to contribute
7. **License** — license type

## Rules
- Never use filler text or lorem ipsum
- Be concise but complete
- Use code blocks with language identifiers
- Include real examples, not abstract ones
